productionprofiles.profileprocessor.ProcessorGasCondensate#

class productionprofiles.profileprocessor.ProcessorGasCondensate(fc: ProfileFile, downtime: int | float | list[tuple] = None)#

Bases: Processor

Calculate, aggregate and resample reservoir production profiles for Gas-Condensate systems. Different schemes can be used to perform aggregation and resampling operations:

  1. drill center name

  2. flow loop name

  3. reservoir name

  4. service type (i.e., production and injection TOTALS)

  5. wells (i.e, no aggregation)

__init__(fc: ProfileFile, downtime: int | float | list[tuple] = None) → None#

Constructor of ProcessorGasCondensate class.

Parameters:

  • fc – Production profile mapping and data object.

  • downtime – Production downtime (expressed in fraction). If a value (float or int) is given, the same downtime is applied to all the entire production profile years. If a list of tuples is passed, the applied downtime is function of the production year(s): - 1st tuple element: downtime value - 2nd tuple element: number of years downtime is applied to. If set to None, no downtime is applied.

Returns:

None

Raises:

TypeError – if fc is not a ProfileFile class instance.

Examples

  1. downtime=0.05 => A 5% downtime is applied to the entire production profile.

  2. downtime=[(0.09, 4), (0.12, 27)] => 9% downtime is applied during the first 4 years, then 12% downtime is applied during the next 27 years.

  3. downtime=[(0.09, 1), (0.10, 1), (0.11, 1), (0.12, 1), (0.13, 1)] => 9% downtime is applied during the first year, then the downtime value increases by one unit during the next 4 years.

aggregate(by: str = 'W') → DataFrame#

Aggregate profile data per drill center, loop, reservoir or service type.

Parameters:

by –

{“DC”, “FL”, “RE”, “TYPE”, “W”} Default “W”. Scheme used for data aggregation:

  • ”FL” : flow loop name

  • ”DC” : drill center name

  • ”RE” : reservoir name

  • ”TYPE” : service type (i.e., production and injection TOTALS)

  • ”W” : no aggregation (all well properties included)

Returns:

Aggregated data. Results are also saved in their corresponding data attribute instance, depending on the chosen aggregation scheme:

  1. ”DC” -> “data_per_dc”

  2. ”FL” -> “data_per_fl”

  3. ”RE” -> “data_per_re”

  4. ”TYPE” -> “data_per_type”

  5. ”W” -> “data_per_well”

Raises:

AggregationError – if aggregation variable (by) has no corresponding value assigned in the ProfileFile mapping dataframe.

calculate(use_fbhp=False) → None#

Calculate additional variables needed as simulation model (e.g., OLGA, LedaFlow) inputs:

  1. Phase ratios (e.g, GOR/WC, CGR/WGR, etc.)

  2. Standard liquid flow rate (QLST)

  3. Reservoir pressure (Pres) if use_fbhp set to True.

Parameters:

use_fbhp – If set to True, the reservoir pressure is calculated based on the flowing bottomhole pressure, liquid (or gas) PI and standard volume flow rate. If False, the reservoir pressure is retained.

Notes

  • gas phase : Pres = QGST / PIG + FBHP

  • liquid phase: Pres = QLST / PIL + FBHP

resample(freq: str = 'Y', by: str = 'W', **kwargs) → DataFrame#

Resample time series data using a frequency conversion. Resampled data are time averaged based on the chosen frequency.

Parameters:

  • freq – (DateOffset, Timedelta or str) Default “Y”. Offset string or object representing target conversion (see example section or Pandas library for more details).

  • by –

    Default “W” {“DC”, “FL”, “RE”, “TYPE”, “W”} Data used for resampling.

    • ”DC” : Aggregated data per drill center

    • ”FL” : Aggregated data per loop

    • ”RE” : Aggregated data per reservoir

    • ”TYPE” : Aggregated data per service type

    • ”W” : Non-aggregated individual well data

  • **kwargs – All additional keyword arguments are passed to the pandas.resample() call.

Returns:

Resampled data. Results are also saved in their corresponding data attribute instance, depending on the chosen data_type identifier:

  • ”DC” -> “data_per_dc”

  • ”FL” -> “data_per_fl”

  • ”RE” -> “data_per_re”

  • ”TYPE” -> “data_per_type”

  • ”W” -> “data_per_well”

Examples

frequency: Y (Year end); M (Month end); Q (Quarter end). For a more exhaustive list of frequencies, see “Offset aliases” subsection at: https://pandas.pydata.org/pandas-docs/stable/user_guide/timeseries.html

    • ”RE” : Aggregated data per reservoir

    • ”TYPE” : Aggregated data per service type

    • ”W” : Non-aggregated individual well data

Returns:

Resampled data. Results are also saved in their corresponding data attribute instance, depending on the chosen data_type:

  • ”DC” -> “data_per_dc”

  • ”FL” -> “data_per_fl”

  • ”RE” -> “data_per_re”

  • ”TYPE” -> “data_per_type”

  • ”W” -> “data_per_well”

Raises:

wrap(sort_by: str = 'LOCS') → dict[slice(<class 'str'>, pandas.core.frame.DataFrame | str, None)]#

Package data into a dict for easy export to an Excel file.

Parameters:

sort_by – {“LOCS”, “VARS”} Default “LOCS” Scheme to sort column results, either by location (“LOCS”) or variable (“VARS”).

Returns:

dict containing sheet names (keys) and data (values). Data may be mapping data with active wells (relationship between wells, drill centers, loops, reservoirs and service types) or aggregated data (data may be also resampled).

Sheets contain:

  • Production data if sheet name starts with “PROD”

  • Injection data if sheet name starts with “INJ”

  • Mapping data if sheet is named “MAPPING”.
